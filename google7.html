Firebase setup (one-time) prerequisite
1
Install Firebase CLI + login
npm install -g firebase-tools firebase logincopied
2
Initialise project (run from repo root)
firebase init # Select: Hosting, Firestore, Storage # Hosting public dir: frontend/build # Single-page app: Yescopied
3
Deploy security rules
firebase deploy --only firestore:rules,storagecopied
4
Create Firestore indexes
firebase deploy --only firestore:indexescopied
5
Seed Kolkata mock data
export GOOGLE_APPLICATION_CREDENTIALS=serviceAccount.json python data/seed_firestore.pycopied
2
React → Firebase Hosting frontend deploy
1
Fill in environment variables
cp frontend/.env.example frontend/.env # Edit .env: add Firebase config + API URLcopied
2
Build the React app
cd frontend npm install npm run build # outputs to frontend/build/copied
3
Deploy to Firebase Hosting
cd .. firebase deploy --only hostingcopied
Live at https://your-project.web.app — CDN-backed, free SSL
4
Update REACT_APP_API_URL for production
REACT_APP_API_URL=https://your-flask-backend.comcopied
Rebuild and redeploy after changing env vars
The rewrites rule in firebase.json sends all paths to index.html — React Router handles /report and /admin client-side. No server config needed.
3
Flask backend — local dev local
1
Create virtualenv and install deps
cd backend python -m venv venv && source venv/bin/activate pip install -r requirements.txtcopied
2
Set environment variables
export GOOGLE_APPLICATION_CREDENTIALS=serviceAccount.json export GCP_PROJECT_ID=your-gcp-project-id export GCP_LOCATION=us-central1 export USE_MOCK_VISION=true # skip real Vision AI locally export USE_MOCK_GEMINI=true # skip real Gemini locallycopied
3
Run Flask dev server
python main.py # Running on http://localhost:5000copied
USE_MOCK_VISION=true and USE_MOCK_GEMINI=true bypass Vertex AI — the mock functions return realistic hardcoded responses so you can develop the full UI flow without GCP credentials.
4
Flask backend — cloud demo production
A
Option A — Google Cloud Run (recommended)
cd backend gcloud run deploy flood-ready-api \ --source . \ --region asia-south1 \ --allow-unauthenticated \ --set-env-vars GCP_PROJECT_ID=your-id,GCP_LOCATION=us-central1copied
Cloud Run auto-scales to zero. ~$0 for a demo. Needs a Dockerfile (see below).
B
Option B — Render.com (simplest for hackathon)
# Push backend/ to GitHub, then on Render: # New Web Service → connect repo # Build command: pip install -r requirements.txt # Start command: gunicorn main:app # Env vars: set GCP_PROJECT_ID, GOOGLE_APPLICATION_CREDENTIALScopied
Free tier works fine for a demo. Cold starts ~30 s.
C
Minimal Dockerfile for Cloud Run
FROM python:3.11-slim WORKDIR /app COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . CMD ["gunicorn","--bind","0.0.0.0:8080","main:app"]copied
For demo day: set REACT_APP_API_URL in the React build to your Cloud Run or Render URL, rebuild, and redeploy to Firebase Hosting. Total pipeline: seed Firestore → deploy Flask → build React → firebase deploy.
